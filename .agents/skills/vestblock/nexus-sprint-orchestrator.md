# Nexus Sprint Orchestrator

Use this skill when running a focused VestBlock build sprint that crosses product, growth, outreach, and operations.

## Inputs

- current business priority
- target funnel or automation area
- relevant product pages
- relevant API routes
- live operational metrics if available

## Goal

Run a tight multi-step sprint where each change is tied to revenue, usability, or operational throughput.

## Sprint order

1. define the bottleneck
2. verify it in the live product or live data
3. make the smallest high-leverage fix
4. verify the new behavior
5. capture the next bottleneck immediately

## Good VestBlock use cases

- improve one sales funnel end to end
- tighten lead ingestion to outreach to send
- launch a new partner or lender path
- upgrade small-business onboarding from request form to guided software feel
- convert new data sources into better contactability or SEO coverage

## Required outputs

- bottleneck statement
- exact shipped change
- verification result
- next sprint target

## Guardrails

- do not drift into broad brainstorming without shipping something
- do not optimize vanity surfaces while revenue or fulfillment blockers remain
- do not claim a flow works without live or data-backed verification
- prefer one completed pass over five half-finished ideas
