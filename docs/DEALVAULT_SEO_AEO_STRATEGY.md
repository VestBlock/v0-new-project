# DealVault SEO / AEO Strategy

## Primary positioning

Lead with:

1. Real estate agreement proof records
2. Referral and JV payout ledger tracking
3. Contractor milestone audit trails

Avoid leading with:

1. Generic blockchain jargon
2. Tokenization
3. Fractional ownership
4. Escrow or custody language

## Best-fit ICPs

1. Wholesalers and co-wholesale operators
2. Investors managing contractors and rehab draws
3. Referral and JV partner networks
4. Creative finance operators with multiple moving agreements

## Core public path

1. `/dealvault`
   Main product page and canonical public entry point
2. `/real-estate-funding`
   Adjacent capital path and cross-sell path
3. `/services`
   Parent comparison page

## Recommended next SEO pages

1. `/dealvault/wholesale-proof-records`
2. `/dealvault/jv-split-tracking`
3. `/dealvault/contractor-milestone-tracking`
4. `/dealvault/referral-payout-ledger`
5. `/dealvault/creative-finance-agreement-proof`

Each page should answer:

1. What problem this solves
2. Who it is for
3. What gets stored on-chain vs off-chain
4. What it does not replace
5. The cleanest next step

## AEO question targets

1. What is a blockchain-backed real estate proof record?
2. How do you track JV splits in a real estate deal?
3. How do you prove when a wholesale agreement was created?
4. How do you track contractor milestones with proof?
5. What should stay off-chain in a real estate workflow?

## Messaging guardrails

Use:

1. Proof record
2. Audit trail
3. Payout ledger
4. Milestone record
5. Operator workflow

Avoid:

1. Crypto-first language
2. Investment promises
3. Legal-enforceability guarantees
4. Compliance claims we cannot substantiate

## CTA ladder

1. Public page CTA: open workspace / request access
2. Secondary CTA: see real estate paths
3. In-product CTA: create deal, create proof, add payout split, add milestone

## Near-term rollout
